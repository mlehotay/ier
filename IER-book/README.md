# IER-book — Book Assembly

This directory contains the **structural and publishing materials** used to assemble the *Informational Experiential Realism (IER)* book.

`IER-book/` is **in-corpus but non-canonical**.
It introduces no new theory and derives all content authority from the canonical IER corpus.

---

## Repository Context

```
IER/          ← Canonical IER corpus (source of truth)
IER-paper/    ← Paper source (non-canonical)
IER-book/     ← Book assembly and publishing
```

* **Canonical chapters:** `IER/`

* **Generated outputs:** `IER-book/build/`

`IER-book/` never contains canonical chapter content.

---

## Assembly Model

The book is a **mechanical assembly**, not an authored document.

**Inputs**

* Canonical chapters from `IER/`
* Preface from `IER-paper/`
* Structural sentinels from `IER-book/src/`

**Outputs**

* Ordered input list (`build/book-input.txt`)
* Rendered formats (PDF / EPUB / HTML)

No content is rewritten or generated.

---

## Build Stages (Conceptual)

1. **Specification** — structure defined in [`IER-book.md`](IER-book.md)
2. **Extraction** — manifest + structure → ordered file list
3. **Validation** — enforce ordering, placement, and scope
4. **Rendering** — renderer consumes validated inputs

Renderers do not define structure.

---

## Notes

* Canonical content lives **only** in `IER/`
* The Preface is source material, not a build artifact
* `build/` is disposable and not committed
* The reader-facing TOC is autogenerated at render time
* Structural filenames encode **order**, not semantics
